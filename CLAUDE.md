# TG Shop — Telegram Mini App каталог взуттєвого бізнесу

## Концепція
Публічний інтернет-каталог бізнесу як Telegram Mini App: будь-хто відкриває застосунок
у Telegram, гортає красивий мінімалістичний каталог наявних товарів, гнучко фільтрує
(тип, бренд, розмір, сезон, колір, стан, ціна) і одним натиском пише продавцю.

Джерело даних — та сама PostgreSQL база `bsstorage`, що й у системі керування бізнесом
(BMS, `~/Desktop/react-fastapi-app`). Цей застосунок **тільки читає** з БД — жодних
записів, жодних мутацій. У каталозі показуються лише наявні (непродані) видимі товари.

## Технічний стек
- **Backend**: Python, FastAPI (порт 8001), SQLAlchemy (text/raw SQL, read-only)
- **Frontend**: React 18 + TypeScript + Vite, без UI-бібліотек (чистий CSS на токенах)
- **Telegram**: офіційний `telegram-web-app.js` (тема, BackButton, MainButton, haptics)
- **БД**: PostgreSQL `bsstorage` — спільна з BMS, схему НЕ змінюємо
- **Фото**: корінь `~/Downloads/Бізнес/Товар`, скануємо РЕКУРСИВНО всі категорії
  (Взуття, Одяг, Сумки, Аксесуари, Інше); URL містить підпапку
  (`/product-images/Одяг/Ф3400_01.JPG`). Конвенція: `<№>_NN` — студійні,
  `<№>_00NN` — реальні, `<№>_defN` — нюанси; донорство студійних фото через
  `products.official_photos_from`, один хоп.

## Ключова бізнес-логіка (узгоджено з BMS — не змінювати самовільно)
- **Наявний товар** = `GREATEST(quantity − sold_count, 0) > 0` І статус журналу
  не у `('Продано','Подаровано','Повернуто')` І `is_lost IS NOT TRUE` І ціна > 0.
  (У `products` НЕМАЄ колонки `is_visible` — видимість керується прапором `is_lost`.)
- Картка каталогу = група рядків з однаковим `productnumber` (ростовка: один номер —
  кілька рядків-розмірів). Сітка групує по `productnumber`, деталь показує всі розміри.
- **sold_count** = замовлення зі статусом Подарунок(7) АБО (Підтверджено(1) І
  Оплачено payment_status_id=1), мінус Повернення(9).
- Номер товару (`productnumber`) — головний публічний ідентифікатор (напр. `Ф1067`).
  Дефіс — НЕ розділювач: `Ф1067-2` — окремий товар (ростовка).
- Службові товари (`???`, `__tmp_rename_…`, без ціни) у каталог не потрапляють.

## Мова та стиль
- Весь інтерфейс, коментарі в коді та документація — українською.
- Дизайн: мінімалістичний монохром (токени `--tg-*`/BMS), автоматична темна тема
  з Telegram, акуратні картки, плавні переходи, нічого зайвого.

## Правила розробки
Усі правила скопійовано з проекту BMS — див. `.cursor/rules/*.mdc`. Головні:
1. Чим менше строк коду, тим краще.
2. Дій як старший розробник.
3. Не зупиняйся, поки функцію не реалізовано повністю.
4. Не роби поспішних висновків — аналізуй.
5. Відповідай коротко.
6. Не видаляй коментарі.
7. Звітуй про поточний стан перед продовженням.
8. Розбивай задачі лише на дійсно потрібні кроки.
10. Ніколи не псуй наявний робочий функціонал.
11. Завжди враховуй цілісність і стабільність проекту.

## Запуск
```bash
./run.sh                # backend :8001 + frontend dev :5173
# або окремо:
cd backend && uvicorn main:app --port 8001 --reload
cd frontend && npm run dev
```
`.env` у корені: DB_NAME/DB_USER/DB_PASSWORD/DB_HOST/DB_PORT, PRODUCT_IMAGES_DIR,
SELLER_TG_USERNAME (кому пише покупець).
